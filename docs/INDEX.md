## Documentation Guidelines

### For Contributors

When adding or updating documentation:

1. **File Naming**
   - Use UPPERCASE for main documentation files
   - Use descriptive names (e.g., DATA_IMPORT_FEATURE.md)
   - Use underscores for multi-word names

2. **Content Structure**
   - Start with a clear title
   - Include table of contents for long documents
   - Use tables for structured information
   - Include code examples where applicable
   - Add "Last Updated" date at the end

3. **Formatting**
   - Use markdown format
   - No emojis in documentation
   - Use proper headings hierarchy
   - Include code blocks with language specification
   - Use tables for comparisons and lists

4. **Status Indicators**
   - Current: Up-to-date documentation
   - Complete: Finished, no longer updated
   - Pending: Work in progress
   - Deprecated: No longer relevant

### Documentation Standards

**Required Sections:**
- Title and overview
- Purpose/objective
- Detailed content
- Examples (where applicable)
- Related documentation links
- Last updated date

**Optional Sections:**
- Prerequisites
- Installation steps
- Configuration
- Troubleshooting
- FAQ
- Version history
